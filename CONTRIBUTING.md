# Contributing to AI Project Template

Thank you for helping improve the Oster Engineering System (OES).

---

## What Belongs Here

Contributions that improve the **template infrastructure**:

- Documentation standards and templates
- Cursor rules, skills, and prompts
- GitHub templates and workflows
- Bootstrap scripts
- Checklists and validation tools

---

## What Does Not Belong Here

- Application code or business logic
- Domain-specific implementations
- Demo apps or placeholder features
- Speculative features not required for the current version

If an idea is valuable but not required now, add it to [.project/PARKING_LOT.md](.project/PARKING_LOT.md).

---

## How to Contribute

1. **Identify the gap** — What did a child project need that the template lacked?
2. **Keep it minimal** — Smallest change that solves the problem.
3. **Follow OES workflow** — Plan before implementation.
4. **Update docs** — CHANGELOG, ROADMAP, or relevant standards.
5. **Use conventional commits** — e.g., `docs: add release checklist template`.

---

## Commit Standards

| Prefix | Use |
|--------|-----|
| `feat:` | New framework capability |
| `fix:` | Fix broken template element |
| `docs:` | Documentation only |
| `chore:` | Maintenance, structure |
| `ci:` | GitHub Actions changes |

---

## Lessons from Child Projects

When a child project completes an MVP, capture:

- What template elements worked well
- What was missing or confusing
- What should be added to the parking lot vs. implemented now

Log these in a framework-improvement issue (template available in Phase 0.4).

---

## Questions

Open an issue with the `question` label, or discuss with the Product Owner directly.
