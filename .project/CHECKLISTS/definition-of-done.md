# Definition of Done

Use this checklist to confirm work is complete before merging or releasing.

---

## Feature Done

A single feature or change is done when:

- [ ] Plan was approved before coding
- [ ] Implementation matches the approved plan
- [ ] Smoke test passes (existing behavior preserved, new behavior works)
- [ ] No architecture drift from approved design
- [ ] Documentation updated if behavior or setup changed
- [ ] [MEMORY.md](../MEMORY.md) updated if project state changed
- [ ] Committed with a conventional commit message
- [ ] PR reviewed and merged (or Owner approved for solo work)

---

## Phase Done

A roadmap phase is done when:

- [ ] All phase deliverables complete
- [ ] Phase success criteria met
- [ ] [ROADMAP.md](../ROADMAP.md) updated (phase marked complete)
- [ ] [MILESTONES.md](../HISTORY/MILESTONES.md) updated
- [ ] Owner validated manually where applicable

---

## MVP Done

The minimum viable product is done when:

- [ ] All MVP scope items complete
- [ ] End-to-end workflow functional
- [ ] Definition of Done met for all MVP features
- [ ] Release tagged (e.g., `v0.1.0` or `v1.0.0`)

Launch, release, and completion checklists are added in Phase 0.6.
