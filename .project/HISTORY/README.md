# Project History

This directory captures how the project evolved over time.

---

## Files

| File | Purpose | Update Frequency |
|------|---------|------------------|
| [SESSION_LOG.md](SESSION_LOG.md) | Rolling log of work sessions | Every session |
| [MILESTONES.md](MILESTONES.md) | Major completed milestones | Each milestone |

---

## Session Log Format

```markdown
## YYYY-MM-DD — [Short session title]

**Phase:** [current phase]
**Worked on:** [summary]
**Completed:** [what shipped]
**Next:** [immediate next step]
```

---

## Milestone Format

```markdown
## [Milestone name] — YYYY-MM-DD

**Phase:** [phase number and name]
**Outcome:** [what was achieved]
**Commit / tag:** [reference if applicable]
```

---

## Rules

1. Append to SESSION_LOG.md — do not rewrite history.
2. Record milestones when phase success criteria are met.
3. Capture lessons learned when something fails — future projects benefit.
